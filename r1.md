# 阶段 6：积分系统 Credits

## 目标

把积分做成真正可靠的账本系统。

## 要完成

```txt
积分账户
积分流水
充值增加积分
AI 调用扣积分
管理员调整积分
失败回滚/退款
余额不足拦截
```

## 积分流水类型

```txt
recharge        充值
consume         消费
refund          退款
admin_adjust    管理员调整
gift            赠送
freeze          冻结
unfreeze        解冻
```

## 关键规则

```txt
所有余额变化必须写流水
不能只改 balance
流水要记录 balanceAfter
扣积分前必须检查余额
AI 调用失败时要退还积分，或者先预估后结算
```

## 核心接口

```txt
GET  /credits/account
GET  /credits/transactions
POST /admin/users/:id/credits/adjust
```

## 验收标准

```txt
用户余额准确
每次余额变化都有流水
管理员调整积分有记录
AI 调用扣费有记录
订单充值加积分有记录
余额不足无法调用 AI
```

# 最重要的开发顺序

你不要从 OIDC、微服务、复杂权限开始。

你应该按这个顺序：

```txt
1. 跑起来
2. 注册登录
3. 用户资料
4. 积分账户
5. 订单
6. 模拟支付
7. AI 调用扣积分
8. 后台查看和管理
9. 安全增强
10. 文档和部署
```

这样你不会卡死在架构里。

# 一句话总结

第一版就做：

```txt
一个 monorepo
三个应用：web / admin / api
三个公共包：db / shared / ui
八个核心模块：auth / users / credits / orders / payments / ai / admin / dashboard
```

先把它做成一个**能注册、能充值、能扣积分、能看后台、能接 AI 的完整闭环**。
这套东西比单纯用户系统更好卖。
