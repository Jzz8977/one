一、整体架构总览

这个项目本质上是一个 "前后端分离 + 数据库 + 缓存 + Nginx 反代" 的经典 SaaS 架构。请求从用户浏览器进来后，先到 Nginx，再分发到前端静态资源或后端 API。

简单理解：你写的 React 项目最终是一堆 .html/.js/.css 静态文件，由 Nginx 提供给浏览器。 浏览器调用 /api/xxx 接口时，Nginx 把请求转发到 NestJS 后端，后端再去查数据库、Redis、调 AI。

三、前端结构（你熟悉的部分）

📁 apps/web — 用户端

  • React 18 + Vite
  • TailwindCSS + 共享 UI 组件库
  • 开发端口：5173
  • 构建产物：apps/web/dist → 静态 HTML/JS/CSS

📁 apps/admin — 管理后台

  • 结构和 web 几乎一样
  • 开发端口：5174
  • 路由前缀通常是 /admin

📦 packages/ — 三端共享代码

包作用
packages/ui共享按钮、表格、弹窗等组件（你两个 React 项目都用）
packages/shared共享类型 / Zod schema / 常量（前后端都用同一份）
packages/dbPrisma schema 和迁移文件（数据库定义）
monorepo 的好处：前后端共享类型。后端定义"订单"的字段，前端能直接 import，类型永远同步，不会再出现"接口字段变了前端不知道"的问题。

四、后端 NestJS 配置

4.1 NestJS 是什么？

NestJS 是一个 Node.js 的服务端框架，写法跟 Angular 很像（用装饰器 @Controller、@Get）。可以理解为"后端版本的 Angular"——约定大于配置，自带依赖注入和模块化。

五、PostgreSQL 数据库

5.1 为什么用 PostgreSQL 而不是 MySQL？

维度PostgreSQLMySQL
JSON 支持✅ 原生 JSONB，可索引JSON 类型有限
事务一致性✅ 更严格一般
SaaS 适配✅ 行级安全、丰富类型需要额外封装
云服务支持阿里云 RDS、腾讯云、Supabase、Neon同样支持

5.2 Prisma 是怎么用的？

你不需要写 SQL，写 JS/TS 就行：

// 在 packages/db/prisma/schema.prisma 定义表
model User {
  id        String   @id @default(cuid())
  email     String   @unique
  password  String
  credits   Int      @default(0)
  createdAt DateTime @default(now())
}

// 在 NestJS 里使用
const user = await prisma.user.findUnique({ where: { email } });
await prisma.user.update({
  where: { id },
  data: { credits: { decrement: 10 } }
});

5.3 数据库迁移流程

1
改 schema.prisma — 加字段或加表
2
pnpm db:migrate — 生成迁移 SQL 文件到 packages/db/prisma/migrations/
3
pnpm db:generate — 生成 TS 类型，前后端都能用
4
上线时 — 执行 prisma migrate deploy 应用到生产数据库

六、Redis 配置与作用

6.1 Redis 是什么？

Redis = 内存里的 KV 数据库，读写超快（微秒级）。你可以把它想象成一个全局的 Map<string, value>，但是跨进程、跨机器、能持久化。

6.2 这个项目里 Redis 做什么？

用途为什么用 Redis 而不用数据库
限流（Throttler）每个用户每分钟最多 60 次请求，需要"快速读 + 自动过期"
Refresh Token 黑名单退出登录后让 token 失效，Redis 自带过期时间
验证码5 分钟过期，存数据库浪费
热数据缓存用户信息、套餐列表等读多写少的数据
类比：PostgreSQL 像硬盘——存得多、可靠、慢一点。Redis 像内存——存得少、超快、断电会丢（除非开持久化）。 所以"关键数据进 PostgreSQL，临时/高频数据进 Redis"。

6.3 配置

# .env 中只有一行
REDIS_URL="redis://localhost:6379"

# 上线时如果是云 Redis（有密码）：
REDIS_URL="redis://default:你的密码@redis.xxx.aliyuncs.com:6379"

七、Docker 与 docker-compose

7.1 Docker 是什么？

Docker 就是"把整套服务打包成一个集装箱"。集装箱里包括代码、Node.js 版本、依赖、操作系统。 丢到任何一台装了 Docker 的机器上，docker compose up 一下就能跑起来。再也不会出现"我电脑能跑你电脑跑不起来"。

7.4 常用 Docker 命令

# 构建所有镜像
docker compose build

# 启动所有服务（-d 后台运行）
docker compose up -d

# 查看运行状态
docker compose ps

# 查看某个服务的日志（最有用！排查 bug 必备）
docker compose logs -f api
docker compose logs -f nginx

# 重启某个服务
docker compose restart api

# 停止全部
docker compose down

# 停止并删除数据卷（⚠️ 会删数据库！）
docker compose down -v

# 进入 api 容器执行命令（比如运行 Prisma 迁移）
docker compose exec api sh

8.2 详细步骤拆解

1
购买服务器
推荐配置：2核 4G / 60G SSD / 5M 带宽。系统选 Ubuntu 22.04 LTS 或 Debian 12。
2
购买域名 + 备案
国内服务器（阿里云/腾讯云）必须 ICP 备案，大约需要 15-20 天。如果想快，买香港或海外服务器免备案。
3
域名解析
登录域名商后台，添加 A 记录：@ 和 www 都指向服务器公网 IP。
4
SSH 登录 + 装环境 
ssh root@你的服务器IP
# 装 Docker
curl -fsSL https://get.docker.com | bash
# 装 git
apt update && apt install -y git
# 装 Node (用来生成密钥等小工具)
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
5
拉代码 + 配置 .env 
cd /opt
git clone 你的仓库地址 ai-saas
cd ai-saas
cp .env.example .env
vim .env  # 重点改：JWT 密钥、OPENAI_API_KEY、WECHAT_*、DATABASE_URL
6
启动服务 
docker compose build
docker compose up -d
docker compose ps  # 确认 6 个服务都在 Up 状态
7
数据库初始化 
docker compose exec api pnpm --filter @app/db deploy   # 跑迁移
docker compose exec api pnpm --filter @app/db seed     # 种子数据
8
HTTPS 证书
最简单方法：用 Caddy 替代 Nginx，自动 HTTPS。或者用 certbot 给 Nginx 申请 Let's Encrypt 免费证书（每 90 天自动续期）。
9
微信支付回调地址
登录商户平台，把 notify_url 改成 https://你的域名/api/payments/wechat/notify。

9.4 数据库要不要用云 RDS？

建议：
  • 用户少（< 1000）：PostgreSQL 跑在同一台 ECS 上，省钱（compose 默认就是这样）
  • 用户多 / 数据重要：用云 RDS PostgreSQL（自动备份、自动主从、不怕服务器挂）
  • 云 Redis 同理：小流量自建，大流量上云
