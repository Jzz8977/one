# 接入 Google / 微信 第三方登录方案

> 目标：让 OAuth 登录后的用户**和邮箱注册用户长得一模一样** —— 同样有 `User` / `UserProfile` / `CreditAccount` / `UserRole` / `gift` 流水，同样发我们自己的 JWT 双 Token，登录后能正常调 `/api/auth/me`、扣积分、走充值流程。

---

## 1. 总体思路

OAuth 三方登录的本质：

```
浏览器 → 三方授权页 → 拿到三方 code
       → 后端用 code 换 access_token + openid/email
       → 后端在我们自己的 User 表里 find-or-create 用户
       → 后端签发我们自己的 JWT (跟邮箱登录走同一条 issueSession 路径)
       → 前端拿到我们的 accessToken/refreshToken，后面流程跟邮箱登录完全一样
```

**关键设计原则**：
- 第三方账号不直接存进 `User` 表，而是用独立的 `UserIdentity` 表关联（标准做法，方便一个用户绑多个三方账号）
- 复用 `auth.service.ts` 里的 `issueSession()` —— 这是邮箱登录、注册、刷新都在用的统一发证函数
- 复用注册时的事务模板：建 User + UserProfile + CreditAccount + UserRole + 写 `gift` 流水

### 2.1 新增 `UserIdentity` 表 + 改 `User`

编辑 `packages/db/prisma/schema.prisma`：

```prisma
model User {
  id               String   @id @default(cuid())
  email            String?  @unique             // ⚠️ 改成可空：微信登录拿不到 email
  phone            String?  @unique
  passwordHash     String?                      // ⚠️ 改成可空：OAuth 用户没密码
  status           String   @default("active")
  failedLoginCount Int      @default(0)
  lockedUntil      DateTime?
  lastLoginAt      DateTime?
  createdAt        DateTime @default(now())
  updatedAt        DateTime @updatedAt

  profile        UserProfile?
  sessions       Session[]
  roles          UserRole[]
  creditAccount  CreditAccount?
  creditTxs      CreditTransaction[]
  orders         Order[]
  payments       PaymentRecord[]
  aiUsageLogs    AiUsageLog[]
  apiKeys        ApiKey[]
  adminLogs      AdminLog[]
  identities     UserIdentity[]                 // ⚠️ 新增反向关联

  @@index([status])
  @@index([createdAt])
}

// 新增表：一个用户可以绑多个第三方账号
model UserIdentity {
  id             String   @id @default(cuid())
  userId         String
  provider       String   // "google" | "wechat" | "wechat_mp" | "github" ...
  providerUserId String   // Google sub / 微信 unionid 或 openid
  email          String?  // 三方返回的邮箱，仅作展示，不可信
  nickname       String?
  avatar         String?
  rawProfile     Json?    // 三方原始返回，方便排查
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([provider, providerUserId])          // 同一个三方账号只能绑一次
  @@index([userId])
}
```

## 3. 通用 OAuth 流程图

```
[前端] 点 "用 Google 登录"
    ↓
[前端] window.location = "https://accounts.google.com/o/oauth2/v2/auth?..."
    ↓
[Google] 用户同意 → 重定向到 https://你的域名/api/auth/oauth/google/callback?code=xxx&state=xxx
    ↓
[后端 callback]
    1) 校验 state（防 CSRF）
    2) code → access_token （POST 三方 token endpoint）
    3) access_token → userInfo （GET 三方 userinfo endpoint）
    4) findOrCreateByIdentity('google', sub, profile)
    5) issueSession(user.id, user.email)
    ↓
[后端] 302 重定向回前端：https://你的前端/login/callback#accessToken=xxx&refreshToken=yyy
    ↓
[前端 LoginCallback 页] 解析 hash → 存入 zustand → 跳 /dashboard
```

### 4.1 `oauth.service.ts` —— 核心逻辑（与 provider 无关）

```typescript
// apps/api/src/modules/oauth/oauth.service.ts
import { Injectable } from '@nestjs/common';
import { PrismaService } from '../../prisma/prisma.service';
import { AuthService } from '../auth/auth.service';
import { ROLES, USER_STATUS } from '@app/shared';
import { loadEnv } from '../../config/env';

export interface OAuthProfile {
  provider: 'google' | 'wechat' | 'wechat_mp';
  providerUserId: string;       // Google: sub / WeChat: unionid 或 openid
  email?: string | null;
  emailVerified?: boolean;
  nickname?: string | null;
  avatar?: string | null;
  raw?: any;
}

@Injectable()
export class OAuthService {
  constructor(
    private prisma: PrismaService,
    private auth: AuthService,        // 复用 issueSession
  ) {}

  /**
   * 三方 profile → 我们的 User，三种情况：
   *   A. identity 已存在 → 直接返回对应 user（老用户登录）
   *   B. identity 不存在但 email 命中已有 user → 自动绑定（同邮箱合并）
   *   C. 都没有 → 新建 User + Identity + Profile + CreditAccount + gift 流水
   */
  async findOrCreate(profile: OAuthProfile, meta: { ip?: string; ua?: string }) {
    const env = loadEnv();

    // A. identity 命中
    const identity = await this.prisma.userIdentity.findUnique({
      where: {
        provider_providerUserId: {
          provider: profile.provider,
          providerUserId: profile.providerUserId,
        },
      },
      include: { user: true },
    });
    if (identity) {
      await this.prisma.user.update({
        where: { id: identity.userId },
        data: { lastLoginAt: new Date() },
      });
      return this.auth.issueForUser(identity.user.id, identity.user.email ?? '', meta);
    }

    // B. email 已存在 → 绑定
    if (profile.email && profile.emailVerified) {
      const existing = await this.prisma.user.findUnique({ where: { email: profile.email } });
      if (existing) {
        await this.prisma.userIdentity.create({
          data: {
            userId: existing.id,
            provider: profile.provider,
            providerUserId: profile.providerUserId,
            email: profile.email,
            nickname: profile.nickname,
            avatar: profile.avatar,
            rawProfile: profile.raw,
          },
        });
        await this.prisma.user.update({
          where: { id: existing.id },
          data: { lastLoginAt: new Date() },
        });
        return this.auth.issueForUser(existing.id, existing.email ?? '', meta);
      }
    }

    // C. 新建
    const role = await this.prisma.role.upsert({
      where: { name: ROLES.USER },
      update: {},
      create: { name: ROLES.USER, description: '普通用户', isSystem: true },
    });

    const giftSetting = await this.prisma.systemSetting.findUnique({
      where: { key: 'auth.default_register_credits' },
    });
    const initialCredits = Number(giftSetting?.value ?? env.DEFAULT_REGISTER_CREDITS);

    const user = await this.prisma.$transaction(async (tx) => {
      const u = await tx.user.create({
        data: {
          email: profile.email ?? null,
          status: USER_STATUS.ACTIVE,
          lastLoginAt: new Date(),
          profile: {
            create: {
              nickname: profile.nickname ?? `${profile.provider}_${profile.providerUserId.slice(0, 6)}`,
              avatar: profile.avatar,
            },
          },
          creditAccount: { create: { balance: initialCredits } },
          roles: { create: { roleId: role.id } },
          identities: {
            create: {
              provider: profile.provider,
              providerUserId: profile.providerUserId,
              email: profile.email,
              nickname: profile.nickname,
              avatar: profile.avatar,
              rawProfile: profile.raw,
            },
          },
        },
      });
      if (initialCredits > 0) {
        await tx.creditTransaction.create({
          data: {
            userId: u.id,
            type: 'gift',
            amount: initialCredits,
            balanceAfter: initialCredits,
            remark: `${profile.provider} 注册赠送`,
          },
        });
      }
      return u;
    });

    return this.auth.issueForUser(user.id, user.email ?? '', meta);
  }
}
```

> ⚠️ `issueSession` 现在是 `AuthService` 里的 **private** 方法。给它加一个 public 包装：

### 4.2 `google.service.ts`

```typescript
// apps/api/src/modules/oauth/google.service.ts
import { Injectable } from '@nestjs/common';
import axios from 'axios';
import { loadEnv } from '../../config/env';
import type { OAuthProfile } from './oauth.service';

@Injectable()
export class GoogleService {
  buildAuthUrl(state: string) {
    const env = loadEnv();
    const params = new URLSearchParams({
      client_id: env.GOOGLE_CLIENT_ID,
      redirect_uri: env.GOOGLE_REDIRECT_URI,
      response_type: 'code',
      scope: 'openid email profile',
      state,
      access_type: 'offline',
      prompt: 'consent',
    });
    return `https://accounts.google.com/o/oauth2/v2/auth?${params}`;
  }

  async exchange(code: string): Promise<OAuthProfile> {
    const env = loadEnv();
    // 1. code → access_token + id_token
    const tokenResp = await axios.post(
      'https://oauth2.googleapis.com/token',
      new URLSearchParams({
        code,
        client_id: env.GOOGLE_CLIENT_ID,
        client_secret: env.GOOGLE_CLIENT_SECRET,
        redirect_uri: env.GOOGLE_REDIRECT_URI,
        grant_type: 'authorization_code',
      }),
      { headers: { 'Content-Type': 'application/x-www-form-urlencoded' } },
    );
    const accessToken = tokenResp.data.access_token as string;

    // 2. access_token → userinfo
    const userResp = await axios.get('https://www.googleapis.com/oauth2/v3/userinfo', {
      headers: { Authorization: `Bearer ${accessToken}` },
    });
    const u = userResp.data;
    return {
      provider: 'google',
      providerUserId: u.sub,         // Google 唯一 ID
      email: u.email,
      emailVerified: u.email_verified === true,
      nickname: u.name,
      avatar: u.picture,
      raw: u,
    };
  }
}
```

### 4.3 `wechat.service.ts` —— 网站应用扫码登录

```typescript
// apps/api/src/modules/oauth/wechat.service.ts
import { Injectable } from '@nestjs/common';
import axios from 'axios';
import { loadEnv } from '../../config/env';
import type { OAuthProfile } from './oauth.service';

@Injectable()
export class WechatService {
  // PC 扫码：open.weixin.qq.com 网站应用
  buildAuthUrl(state: string) {
    const env = loadEnv();
    const params = new URLSearchParams({
      appid: env.WECHAT_OPEN_APP_ID,
      redirect_uri: env.WECHAT_OPEN_REDIRECT_URI,
      response_type: 'code',
      scope: 'snsapi_login',
      state,
    });
    return `https://open.weixin.qq.com/connect/qrconnect?${params}#wechat_redirect`;
  }

  async exchange(code: string): Promise<OAuthProfile> {
    const env = loadEnv();
    // 1. code → access_token + openid
    const tokenResp = await axios.get('https://api.weixin.qq.com/sns/oauth2/access_token', {
      params: {
        appid: env.WECHAT_OPEN_APP_ID,
        secret: env.WECHAT_OPEN_APP_SECRET,
        code,
        grant_type: 'authorization_code',
      },
    });
    if (tokenResp.data.errcode) throw new Error(`wechat token error: ${tokenResp.data.errmsg}`);
    const { access_token, openid, unionid } = tokenResp.data;

    // 2. access_token + openid → userinfo
    const userResp = await axios.get('https://api.weixin.qq.com/sns/userinfo', {
      params: { access_token, openid, lang: 'zh_CN' },
    });
    if (userResp.data.errcode) throw new Error(`wechat user error: ${userResp.data.errmsg}`);
    const u = userResp.data;

    return {
      provider: 'wechat',
      // ⚠️ 优先用 unionid（同一开放平台下跨应用唯一），降级用 openid
      providerUserId: unionid || openid,
      email: null,                  // 微信不给 email
      emailVerified: false,
      nickname: u.nickname,
      avatar: u.headimgurl,
      raw: u,
    };
  }
}
```

### 4.4 `oauth.controller.ts`

```typescript
// apps/api/src/modules/oauth/oauth.controller.ts
import { Controller, Get, Query, Req, Res } from '@nestjs/common';
import { randomBytes } from 'crypto';
import type { Request, Response } from 'express';
import { Public } from '../../common/decorators/public.decorator';
import { OAuthService } from './oauth.service';
import { GoogleService } from './google.service';
import { WechatService } from './wechat.service';
import { loadEnv } from '../../config/env';

@Controller('auth/oauth')
export class OAuthController {
  constructor(
    private oauth: OAuthService,
    private google: GoogleService,
    private wechat: WechatService,
  ) {}

  // ===== Google =====
  @Public()
  @Get('google')
  google_start(@Res() res: Response) {
    const state = randomBytes(16).toString('hex');
    res.cookie('oauth_state', state, { httpOnly: true, sameSite: 'lax', maxAge: 5 * 60_000 });
    return res.redirect(this.google.buildAuthUrl(state));
  }

  @Public()
  @Get('google/callback')
  async google_callback(
    @Query('code') code: string,
    @Query('state') state: string,
    @Req() req: Request,
    @Res() res: Response,
  ) {
    if (!code || state !== (req as any).cookies?.oauth_state) {
      return res.redirect(this.failUrl('state_mismatch'));
    }
    res.clearCookie('oauth_state');
    try {
      const profile = await this.google.exchange(code);
      const tokens = await this.oauth.findOrCreate(profile, {
        ip: req.ip,
        ua: req.headers['user-agent'],
      });
      return res.redirect(this.successUrl(tokens));
    } catch (e: any) {
      return res.redirect(this.failUrl(e.message ?? 'oauth_error'));
    }
  }

  // ===== WeChat (网站扫码) =====
  @Public()
  @Get('wechat')
  wechat_start(@Res() res: Response) {
    const state = randomBytes(16).toString('hex');
    res.cookie('oauth_state', state, { httpOnly: true, sameSite: 'lax', maxAge: 5 * 60_000 });
    return res.redirect(this.wechat.buildAuthUrl(state));
  }

  @Public()
  @Get('wechat/callback')
  async wechat_callback(
    @Query('code') code: string,
    @Query('state') state: string,
    @Req() req: Request,
    @Res() res: Response,
  ) {
    if (!code || state !== (req as any).cookies?.oauth_state) {
      return res.redirect(this.failUrl('state_mismatch'));
    }
    res.clearCookie('oauth_state');
    try {
      const profile = await this.wechat.exchange(code);
      const tokens = await this.oauth.findOrCreate(profile, {
        ip: req.ip,
        ua: req.headers['user-agent'],
      });
      return res.redirect(this.successUrl(tokens));
    } catch (e: any) {
      return res.redirect(this.failUrl(e.message ?? 'oauth_error'));
    }
  }

  private successUrl(tokens: { accessToken: string; refreshToken: string; expiresIn: number }) {
    const env = loadEnv();
    const hash = new URLSearchParams({
      accessToken: tokens.accessToken,
      refreshToken: tokens.refreshToken,
      expiresIn: String(tokens.expiresIn),
    });
    return `${env.WEB_BASE_URL}/login/callback#${hash.toString()}`;
  }
  private failUrl(reason: string) {
    const env = loadEnv();
    return `${env.WEB_BASE_URL}/login?reason=${encodeURIComponent(reason)}`;
  }
}
```

## 6. 三方平台注册步骤

### 6.1 Google Cloud Console
1. 访问 https://console.cloud.google.com → 创建项目
2. 「APIs & Services → OAuth consent screen」配置应用名称、邮箱、scope（`openid email profile`）
3. 「Credentials → Create Credentials → OAuth client ID」
   - Application type：**Web application**
   - Authorized redirect URIs：
     - 开发：`http://localhost:4000/api/auth/oauth/google/callback`
     - 生产：`https://your-domain.com/api/auth/oauth/google/callback`
4. 拿到 `Client ID` / `Client Secret` → 填到 `.env`

### 6.2 微信开放平台（PC 扫码）
1. 注册 https://open.weixin.qq.com （需企业认证，¥300/年）
2. 「管理中心 → 网站应用 → 创建网站应用」
3. 填授权回调域：`your-domain.com`（**只填域名，不带协议和路径**）
4. 应用审核通过后拿到 `AppID` / `AppSecret` → 填到 `.env`

> 💡 如果是面向微信内部 H5（公众号网页）或小程序，授权流程稍有不同：
> - **公众号网页授权**：用 `snsapi_userinfo`，需公众号绑定服务号 + 在公众号后台配置授权回调域
> - **小程序**：用 `wx.login()` 获取 code → 后端 `code2Session` → 拿 openid/unionid，不走浏览器跳转
> 这两种场景把对应的 service 类再写一个就行，复用同一个 `OAuthService.findOrCreate`。

### 7.2 新增回调页 `apps/web/src/pages/LoginCallback.tsx`

```tsx
import { useEffect } from 'react';
import { useNavigate } from 'react-router-dom';
import { useAuthStore } from '../store/auth';
import { useToast } from '@app/ui';

export function LoginCallbackPage() {
  const navigate = useNavigate();
  const toast = useToast();
  const setTokens = useAuthStore((s) => s.setTokens);

  useEffect(() => {
    const hash = window.location.hash.slice(1);
    const params = new URLSearchParams(hash);
    const accessToken = params.get('accessToken');
    const refreshToken = params.get('refreshToken');
    if (accessToken && refreshToken) {
      setTokens(accessToken, refreshToken);
      // 清掉地址栏的 token
      window.history.replaceState({}, '', '/login/callback');
      toast.success('登录成功');
      navigate('/dashboard', { replace: true });
    } else {
      toast.error('登录失败');
      navigate('/login', { replace: true });
    }
  }, [navigate, setTokens, toast]);

  return <div className="flex min-h-screen items-center justify-center">正在登录...</div>;
}
```

## 8. 流程对照表

| 步骤 | 邮箱注册（现有） | Google 登录（新增） | 微信登录（新增） |
|---|---|---|---|
| 1. 入口 | 填邮箱密码 | 跳 Google 授权页 | 跳微信扫码页 |
| 2. 拿到三方凭证 | — | code → access_token | code → access_token + openid |
| 3. 拿用户信息 | 邮箱密码 | userinfo（email、sub） | userinfo（unionid、nickname） |
| 4. 建库（首次） | User+Profile+Account+Role+gift | 同左 + UserIdentity | 同左 + UserIdentity |
| 5. 发证 | `issueSession()` | **同一个** `issueSession()` | **同一个** `issueSession()` |
| 6. 前端拿到 token | JSON 返回 | URL hash 回跳 | URL hash 回跳 |
| 7. 后续接口 | 完全一致 | 完全一致 | 完全一致 |

## 9. 容易踩坑的点

| 坑 | 说明 |
|---|---|
| `passwordHash` 不能为空 | 必须在迁移里改成可空，否则 OAuth 用户无法创建 |
| `email` 不能为空 | 微信不给 email，必须改成可空，并去掉密码登录里的 `findUnique({where:{email}})` 假定 |
| 微信不返回 email | 不能用 email 自动绑定。如果想合并账号，要让用户登录后手动绑邮箱 |
| state 没校验 | 必有 CSRF 风险，本文用 httpOnly cookie 存 state |
| openid vs unionid | **永远优先 unionid**。同一微信号在不同应用下 openid 不同，但 unionid 相同 |
| redirect_uri 不一致 | Google/微信都严格校验，开发和生产分别加白名单 |
| 回调把 token 放 URL | 用 `#hash`（不会带到日志/Referer）比 `?query` 安全得多 |
| 微信开放平台需要企业认证 | 个人开发者无法走 PC 扫码，得用公众号或小程序绕 |
| 国内服务器 + Google | 服务器需要能访问 google.com。如果在国内 ECS，要走代理或选海外服务器 |
| 登录后没积分 | `findOrCreate` 里 C 分支必须复用 gift 流水逻辑，不然新用户余额对，但没流水会影响后台对账 |

## 10. 后续可做的增强

- **绑定/解绑页面**：在 `/profile` 页加"已绑定账号"列表（从 `userIdentity` 查），允许登录用户额外绑定 Google/微信
- **强制邮箱绑定**：微信用户首次登录后弹一个"补充邮箱"步骤，方便后续找回账号
- **登录日志**：在 `AdminLog` 或新增表里记 `login_method=google/wechat/password`
- **关闭密码登录**：在 `system_setting` 加 `auth.password_login_enabled` 开关
- **抽象成 Strategy**：当三方多到 4 个以上，把每个 provider 封成 `OAuthStrategy` 接口（`buildAuthUrl` + `exchange`），controller 用 `:provider` 参数路由
- **passport-google-oauth20 / passport-wechat**：如果不想手撸，可以接入 `@nestjs/passport` + 对应 passport 策略，少写一些代码但多一层抽象
